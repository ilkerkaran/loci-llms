# loci-llms

Deliverable bundle and preview hosting for the Loci **`llms.txt`** and **`llms-full.txt`** — an LLM-friendly content index for the public corporate website at [lociapp.co.uk](https://lociapp.co.uk), following the [llms.txt proposal](https://llmstxt.org/).

## What's in here

| File | Purpose |
| --- | --- |
| [`llms.txt`](./llms.txt) | Curated markdown index of Loci's public site, targeted at commercial / sales LLM questions. |
| [`llms-full.txt`](./llms-full.txt) | Same index, with the commercial-priority pages (For Councils, Our Platform, LGR, Trust) inlined for single-fetch consumption. Option Y — see `DESIGN.md`. |
| [`robots.txt`](./robots.txt) | Proposed replacement for the current `lociapp.co.uk/robots.txt`. Preserves the HubSpot defaults, adds a `Sitemap:` reference, and declares an AI-crawler allow-list. |
| [`sitemap.xml`](./sitemap.xml) | Preview-only minimal sitemap. Production `sitemap.xml` is auto-generated by HubSpot — see `PUBLISHING.md` for the two entries the agency must add there. |
| [`index.html`](./index.html) | **Preview-only** stub homepage with discovery hooks (`<link rel="alternate">`, footer link). Not for production. Used purely so the GitHub Pages / Netlify preview has a landing page that points LLMs at `/llms.txt`. |
| [`PUBLISHING.md`](./PUBLISHING.md) | Step-by-step handover runbook for the agency that manages `lociapp.co.uk` on HubSpot CMS. |
| [`QA-CHECKLIST.md`](./QA-CHECKLIST.md) | Post-publish verification steps. |
| [`DESIGN.md`](./DESIGN.md) | Short record of the decisions behind this bundle. |

## Test plan (short version)

Three-tier test against the preview URL:

1. **T1 — Control.** Give an LLM the explicit `/llms.txt` URL; must accurately summarise Loci.
2. **T2 — Semi-assisted.** Give the preview root URL and a prompt that mentions "use llms.txt if available"; the LLM should fetch and use the file.
3. **T3 — Pure auto-discovery.** Give only the root URL and a neutral question. Most LLMs in 2026 will not auto-discover `/llms.txt` from a root URL; T3 is informational, not blocking.

See `DESIGN.md` for the full plan and `QA-CHECKLIST.md` for mechanical checks.

## Maintenance

- **On each new council launch** — update `## Councils live on Loci` in `llms.txt` and `llms-full.txt`, and refresh the `lastmod` in `sitemap.xml`.
- **Quarterly** — diff the inlined content in `llms-full.txt` against the live HubSpot pages; resync anything drifted.
- Source of the launched-councils list: PostHog `council-launched` feature flag cross-referenced with the `LocalAuthorities` table in the `councildata` Postgres database.
